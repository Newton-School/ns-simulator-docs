# Ticket Delivery Workflow

## Pre-implementation checks
- Ticket is unblocked.
- Required files and modules are identified.
- Acceptance criteria are clear and testable.

## Delivery steps
- Implement smallest complete behavior first.
- Add missing validation/tests needed for AC.
- Avoid unrelated refactors in the same ticket.

## Completion checklist
- Every AC item is explicitly marked pass/fail.
- Verification commands and outcomes are recorded.
- Known limitations are documented.
